# Security Policy

## Supported Versions

Security fixes apply to the current `main` branch.

## Reporting a Vulnerability

Please do not publish exploit details or private test files in a public issue.
Report suspected vulnerabilities privately through GitHub's security reporting
features when available, or contact the repository owner through GitHub.

Include:

- a short description of the issue
- steps to reproduce
- browser and operating system
- impact and any known workaround

## File Handling

WebTracker parses user-selected module and audio files locally in the browser.
Avoid sharing copyrighted or private module files in public reports. If a file
is required to reproduce a problem, create the smallest possible synthetic test
case or share privately with permission.
